# 🎙️ SoulSpeak v2.0 — Python Flask Backend

Speech Personality Recognition with **real ML models**, **Whisper transcription**, and **Ollama AI coaching**.

---

Models available at: https://huggingface.co/NSankalpa/soulspeak-models
Download and place in ml_models/ before running.


## Architecture

```
Audio File / Mic Recording
        │
        ├─── 🔊 ACOUSTIC BRANCH ────────────────────────────────────
        │    Resample to 16 kHz mono (librosa)
        │    → wav2vec2_personality_best.pt
        │       (Wav2Vec2 backbone + MLP → 5 Big Five scores)
        │
        ├─── 📝 LINGUISTIC BRANCH ──────────────────────────────────
        │    Whisper transcription (openai-whisper)
        │    → bert_personality_best.pt
        │       (BERT backbone + MLP → 5 Big Five scores)
        │
        └─── ⚖️  FUSION (50/50 weighted average, adjustable)
                 → Final OCEAN scores as percentages (0–100)
                 → Coaching insights + recommendations

Chat
  User message → Ollama (local LLM, streaming SSE) → VOXMIND response
```

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

For GPU acceleration:
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
pip install -r requirements.txt
```

### 2. Place your model files

Put both `.pt` files in the project root (same folder as `app.py`):
```
wav2vec2_personality_best.pt
bert_personality_best.pt
```

### 3. Start Ollama

```bash
# Make sure Ollama is running
ollama serve

# Pull a model if you haven't already
ollama pull llama3        # recommended
# or: ollama pull mistral, phi3, qwen2, gemma2, etc.
```

### 4. Run setup (one time only)

```bash
python setup.py
```

### 5. Start the server

```bash
python app.py
```

Open **http://localhost:5000**

## Configuration (environment variables)

| Variable | Default | Description |
|----------|---------|-------------|
| `WAV2VEC2_PATH` | `wav2vec2_personality_best.pt` | Path to Wav2Vec2 model |
| `BERT_PATH` | `bert_personality_best.pt` | Path to BERT model |
| `WHISPER_SIZE` | `base` | Whisper model size: `tiny`, `base`, `small`, `medium` |
| `OLLAMA_URL` | `http://localhost:11434` | Ollama base URL |
| `OLLAMA_MODEL` | *(auto-detected)* | Force a specific Ollama model |
| `ACOUSTIC_WEIGHT` | `0.5` | Wav2Vec2 weight in fusion (0.0–1.0) |
| `PORT` | `5000` | Flask port |

Example:
```bash
WHISPER_SIZE=small ACOUSTIC_WEIGHT=0.6 OLLAMA_MODEL=mistral python app.py
```

## Model Architecture (what the `.pt` files must match)

### `wav2vec2_personality_best.pt`
```
Wav2Vec2Model (facebook/wav2vec2-base OR facebook/wav2vec2-large)
→ Mean pooling over time frames
→ LayerNorm(768)
→ Dropout(0.1)
→ Linear(768 → 256) + GELU
→ Dropout(0.1)
→ Linear(256 → 5)
→ Sigmoid  →  [O, C, E, A, N] in [0, 1]
```

### `bert_personality_best.pt`
```
BertModel (bert-base-uncased OR bert-large-uncased)
→ CLS token hidden state
→ LayerNorm(768)
→ Dropout(0.1)
→ Linear(768 → 256) + GELU
→ Dropout(0.1)
→ Linear(256 → 5)
→ Sigmoid  →  [O, C, E, A, N] in [0, 1]
```

The loader tries both strict and non-strict state dict loading, and auto-detects
`base` vs `large` from the weight shapes. If loading fails it falls back to
a profession-seeded simulation so the app never crashes.

---

## Fallback Behaviour

| Situation | Behaviour |
|-----------|-----------|
| Model files missing | Profession-based simulation |
| Model files present but wrong architecture | Non-strict load (best-effort), then simulation |
| Whisper not installed | No transcription; BERT branch skipped |
| Ollama offline | Chat returns a helpful error message |
| No audio file uploaded | Simulation based on duration + profession |
